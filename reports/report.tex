\section{Introduction and Objective}

Superconducting materials are characterized by a critical temperature above which
their superconducting behavior disappears. Predicting this temperature from
chemical descriptors is a useful statistical learning problem because experimental
measurement is costly, while composition-level information is comparatively easy
to record. In this project, the response variable is
\(Y=\texttt{critical\_temp}\), measured in Kelvin, and the predictors are the 81
numerical physical and chemical covariates supplied in the UCI superconductivity
dataset.

The problem is formulated as supervised regression. Given observations
\((x_i,y_i)\), with \(x_i\in\mathbb{R}^{81}\) and \(y_i\in\mathbb{R}_+\), the
objective is to build a function \(\widehat f\) such that \(\widehat f(x)\)
predicts the critical temperature of a new material as accurately as possible. We
compare a constant baseline, ordinary least squares, Ridge regression, Lasso
regression, k-nearest-neighbor regression, and a histogram-based gradient
boosting regressor. The first four non-baseline methods allow us to connect the
analysis with the course material, while the boosting model is included as an
explicitly explained nonlinear benchmark.

\section{Dataset and Data Audit}

The raw UCI dataset contains 21,263 materials. Each material is described by 81
engineered numerical variables derived from chemical formulae and elemental
properties such as atomic mass, first ionization energy, atomic radius, density,
electron affinity, fusion heat, thermal conductivity, and valence. The engineered
variables include summary statistics such as arithmetic means, weighted means,
geometric means, ranges, entropies, and standard deviations. The target column is
\texttt{critical\_temp}.

Before modeling, exact duplicate rows in the full feature-target table are
removed. This is done before the train-test split, so identical observations
cannot appear once in the training set and once in the test set. The cleaning step
removes 66 exact duplicates and leaves 21,197 observations. No missing values are
present after cleaning, so no imputation policy is needed for this version of the
dataset. The first-stage data audit is saved by the pipeline as JSON and CSV
artifacts under \texttt{reports/tables/}.

The target distribution is right-skewed. The mean critical temperature is
approximately 34.49 K, the median is 20 K, and the maximum is 185 K. This means
that a model can obtain low average error on the many low-temperature materials
while still making scientifically important mistakes in the high-temperature tail.
For that reason, the model comparison is supplemented with residual diagnostics
by target range.

\section{Preprocessing and Leakage Control}

The preprocessing sequence is designed to make the final test-set results
credible. First, the raw UCI feature and target tables are merged and exact full
rows are removed. Second, the cleaned table is split once into a training set and
a held-out test set using an 80/20 split with random seed 42. The held-out test set
is not used for hyperparameter selection.

For OLS, Ridge, Lasso, and k-nearest neighbors, the covariates are standardized
inside a scikit-learn pipeline. The centering and scaling parameters are fit only
on the relevant training fold during cross-validation and only on the full
training set before final test evaluation. This avoids data leakage from the test
set into preprocessing. The tree-based histogram gradient boosting model is not
sensitive to monotone feature scaling, so it is fit without standardization.

The target is transformed during fitting using
\[
Z=\log(1+Y).
\]
Predictions are transformed back to the original Kelvin scale using
\(\widehat Y=\exp(\widehat Z)-1\) before metrics are computed. The transformation
reduces the dominance of the high-temperature tail during fitting and ensures that
model comparisons are still reported in physically interpretable Kelvin units.

\section{Models}

\paragraph{Constant baseline.}
The baseline predicts a constant value learned from the training data. It provides
a minimal reference: a useful model must improve substantially on this baseline.

\paragraph{Ordinary least squares.}
The linear regression model assumes
\(\widehat f(x)=\widehat\beta_0+x^\top\widehat\beta\), where the coefficients are
chosen to minimize the residual sum of squares on the transformed target. OLS is
simple and interpretable, but it can be unstable when predictors are correlated.

\paragraph{Ridge regression.}
Ridge regression adds an \(\ell_2\) penalty to stabilize the linear model:
\[
\min_{\beta_0,\beta}\sum_{i=1}^{n}(z_i-\beta_0-x_i^\top\beta)^2+
\lambda\sum_{j=1}^{p}\beta_j^2.
\]
The penalty shrinks coefficients toward zero without setting them exactly to zero.
The regularization parameter is selected by cross-validation on the training set.

\paragraph{Lasso regression.}
Lasso regression replaces the Ridge penalty by an \(\ell_1\) penalty:
\[
\min_{\beta_0,\beta}\sum_{i=1}^{n}(z_i-\beta_0-x_i^\top\beta)^2+
\lambda\sum_{j=1}^{p}|\beta_j|.
\]
This can set some coefficients exactly to zero, which makes the fitted linear
model more sparse. As with Ridge, the penalty parameter is selected without using
the held-out test set.

\paragraph{k-nearest-neighbor regression.}
k-NN predicts a new response by averaging the responses of the closest training
observations in standardized feature space. It is nonlinear and can adapt to local
structure in the data. The number of neighbors \(k\) is selected by grid search
with cross-validation. This method is powerful here but stores the training set
and can be sensitive to the choice of scaling and distance metric.

\paragraph{Histogram-based gradient boosting.}
The histogram gradient boosting regressor is included as an additional nonlinear
benchmark. It builds an additive model of regression trees, where each new tree is
fit to reduce the current prediction error. This method is not treated as a
course-covered method in the report, so its functioning is stated explicitly. It
is useful for this dataset because interactions between chemical descriptors are
plausible and may not be captured by linear models.

\section{Model Selection and Evaluation Criteria}

Model selection is performed only with the training data. Ridge and Lasso choose
regularization strengths from predefined grids, and k-NN chooses
\(k\in\{3,5,7,11,15,21,31\}\) by five-fold cross-validation. The gradient boosting
hyperparameters are fixed in the configuration file and include a learning rate of
0.06, 350 maximum boosting iterations, 31 maximum leaf nodes, \(\ell_2\)
regularization 0.01, and early stopping on an internal validation fraction.

For observations \((y_i,\widehat y_i)_{i=1}^n\) in the held-out test set, the main
criteria are
\[
\mathrm{MSE}=\frac{1}{n}\sum_{i=1}^n (y_i-\widehat y_i)^2,\qquad
\mathrm{RMSE}=\sqrt{\mathrm{MSE}},
\]
\[
\mathrm{MAE}=\frac{1}{n}\sum_{i=1}^n |y_i-\widehat y_i|,\qquad
R^2=1-\frac{\sum_{i=1}^n (y_i-\widehat y_i)^2}
{\sum_{i=1}^n (y_i-\bar y)^2}.
\]
RMSE is the primary selection criterion because it is expressed in Kelvin and
penalizes large errors strongly. MAE is less sensitive to isolated large errors,
and \(R^2\) measures the fraction of test-set variance explained relative to a
constant predictor.

The nonlinear models outperform the linear family by a large margin. The
k-nearest-neighbor model obtains the lowest RMSE, approximately 10.66 K, the
lowest MAE, approximately 5.67 K, and the highest \(R^2\), approximately 0.904.
The histogram-based gradient boosting model is very close, with RMSE approximately
10.75 K and \(R^2\) approximately 0.902. OLS, Ridge, and Lasso remain around RMSE
18.3 K. This gap suggests that the relationship between the engineered physical
descriptors and the critical temperature contains nonlinear structure or
interactions that linear models do not capture well.

The best selected k-NN model uses \(k=3\), while Ridge selects a small penalty
\(\alpha=0.1\) and Lasso selects \(\alpha=10^{-4}\). The small amount of
regularization selected for the linear models is consistent with the fact that the
linear signal is present but insufficient: shrinkage stabilizes coefficients, yet
it cannot create the nonlinear local behavior captured by k-NN and boosting.

\section{Discussion}

The comparison leads to two main conclusions. First, the nonlinear structure in
the dataset is strong: k-NN and gradient boosting improve the RMSE by more than
7 K compared with the linear methods. Second, the best predictive model is not
necessarily the most interpretable model. Ridge and Lasso summarize the marginal
linear contribution of standardized descriptors and are easier to explain, but
both lose substantial predictive accuracy. k-NN gives the best held-out metrics in
this run, while gradient boosting is nearly as accurate and may be easier to use
in production because prediction does not require searching through all training
observations.

The target transformation also matters. Fitting on \(\log(1+Y)\) reduces the
influence of extremely high target values and helps all methods produce positive,
stable predictions after inverse transformation. The trade-off is that the fitted
objective is not exactly the same as minimizing squared error directly in Kelvin.
This trade-off is acceptable here because final evaluation is always performed on
the original scale and because the transformed target is much less skewed.

\section{Reproducibility}

All main computations are organized as reusable Python modules under \path{src/}.
The scripts \path{scripts/run_eda.py}, \path{scripts/train_models.py}, and
\path{scripts/evaluate_models.py} produce the tables, fitted model artifacts, and
figures used in this report. The configuration file \path{configs/default.yaml}
records the random seed, the train-test split, the cross-validation settings, the
metric list, and model hyperparameter grids. Package versions are managed with
\texttt{uv} and locked in \path{uv.lock}. This organization makes the final
notebook a clean assembly of already-tested pipeline steps rather than the only
place where the analysis exists.

\section{Limitations and Possible Extensions}

The final model is evaluated on a random held-out split, not on an external
experimental dataset. The reported error therefore measures generalization to
similar observations from the same data source, not necessarily to future
materials synthesized under different experimental conditions. In addition, the
analysis uses the engineered UCI descriptors directly and does not reconstruct
chemical features from raw formulae. Some materials with very high critical
temperature are still underpredicted, suggesting that more specialized features,
local models for the high-temperature tail, or uncertainty estimates could be
useful extensions.

Another limitation is interpretability. The strongest model, k-NN, is local and
non-parametric. It explains a prediction through nearby training examples rather
than through global coefficients. For a scientific report, Ridge and Lasso remain
valuable because they provide a simpler linear reference, but their predictive
performance is clearly lower. Future work could add model-agnostic interpretation
tools or compare with additional methods after explaining them carefully, provided
that the course-method distinction is maintained.

\section{Conclusion}

The project defines a supervised regression problem for predicting
\texttt{critical\_temp} from 81 physical and chemical covariates. After duplicate
removal, leakage-controlled preprocessing, cross-validation on the training set,
and final evaluation on a held-out test set, k-nearest-neighbor regression with
\(k=3\) is the recommended model. It achieves RMSE about 10.66 K, MAE about
5.67 K, and \(R^2\) about 0.904. Histogram-based gradient boosting is a very close
nonlinear alternative. Linear regression, Ridge, and Lasso are useful baselines
and course-connected references, but they are not accurate enough to match the
nonlinear methods on this dataset. The final diagnostic analysis shows that the
main remaining challenge is accurate prediction for the high-temperature tail.
